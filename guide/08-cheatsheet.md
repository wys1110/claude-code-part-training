# 8. 치트시트

## 시작과 세션

```bash
claude                         # 대화형 세션 시작
claude "이 프로젝트 설명해줘"  # 초기 요청과 함께 시작
claude --permission-mode plan  # 계획 모드
claude --continue              # 최근 세션 이어가기
claude --resume                # 특정 세션 선택
claude -p "요청"              # 비대화형 실행
```

## 자주 쓰는 세션 명령

```text
/help          사용 가능한 명령 확인
/init          프로젝트 CLAUDE.md 초기화
/memory        지침·메모 확인
/permissions   권한 확인·수정
/resume        이전 세션 불러오기
/clear         현재 대화 컨텍스트 정리
```

## 작업 시작 프롬프트

```text
관련 파일과 테스트를 먼저 찾아 현재 동작을 설명해줘.
아직 수정하지 말고 원인과 변경 계획을 제시해줘.
```

## 구현 프롬프트

```text
합의한 범위만 최소 수정해줘.
기존 public API와 데이터 형식은 유지하고 관련 없는 리팩터링은 하지 마.
```

## 검증 프롬프트

```text
관련 테스트, 린트, 빌드를 실행해줘.
실행한 명령, 결과, 변경 파일, 남은 위험을 요약해줘.
```

## 리뷰 프롬프트

```text
현재 diff만 리뷰해줘.
정확성, 회귀, 보안, 테스트 누락 순으로 중요도와 근거를 알려줘.
```

## Git 프롬프트

```text
현재 변경을 목적별로 요약하고 적절한 커밋 메시지 후보를 만들어줘.
아직 commit이나 push는 하지 마.
```

## 기억할 문장

> **먼저 읽게 하고, 계획을 확인하고, 작게 수정하고, 테스트로 증명한다.**
