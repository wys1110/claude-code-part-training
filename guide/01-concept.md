# 1. Claude Code 개념

<div class="badge-row"><span>난이도: 입문</span><span>권장 시간: 10~15분</span><span>핵심: 에이전트 루프</span></div>

## 한 문장 정의

**Claude Code는 코드베이스를 읽고, 파일을 수정하고, 명령을 실행하며, 결과를 검증하는 에이전트형 개발 도구입니다.**

일반 채팅형 AI가 주로 답변을 생성한다면, Claude Code는 개발 환경 안에서 도구를 사용해 실제 작업을 수행합니다.

| 구분 | 일반 채팅형 AI | Claude Code |
|---|---|---|
| 컨텍스트 | 사용자가 붙여 넣은 내용 중심 | 프로젝트 파일을 필요한 만큼 탐색 |
| 행동 | 설명·코드 제안 | 파일 편집, 명령 실행, Git 작업 |
| 검증 | 사용자가 직접 확인 | 테스트·린트·빌드를 실행해 확인 가능 |
| 지속 설정 | 대화마다 반복 설명 | `CLAUDE.md`, 설정, Skills로 공유 가능 |

## 핵심: 에이전트 루프

Claude Code는 보통 다음 흐름으로 작업합니다.

<div class="flow"><div>1. 컨텍스트 수집</div><div>2. 작업 수행</div><div>3. 결과 검증</div></div>

### 1) 컨텍스트 수집

- 프로젝트 구조와 관련 파일 탐색
- 기존 구현·테스트·규칙 확인
- 모호한 요구사항 식별

### 2) 작업 수행

- 코드와 문서 수정
- 필요한 명령 실행
- 여러 파일 간 일관성 유지

### 3) 결과 검증

- 테스트·린트·빌드 실행
- 변경 diff 확인
- 실패 시 원인을 다시 탐색하고 수정

::: tip 좋은 요청의 기본형
`관련 코드를 먼저 분석하고 → 변경 계획을 제시한 뒤 → 최소 범위로 수정하고 → 테스트로 검증해줘`
:::

## Claude Code가 특히 유용한 일

- 처음 보는 코드베이스 구조 파악
- 오류 메시지와 실패 테스트 기반 디버깅
- 작은 기능 구현과 리팩터링
- 테스트·문서·릴리스 노트 작성
- Git diff 리뷰, 커밋 메시지, PR 초안 작성
- 반복 명령이나 팀 워크플로 자동화

## 사람에게 남는 책임

Claude Code가 실행한다고 해서 결과가 자동으로 정답이 되는 것은 아닙니다.

<ul class="checklist">
<li>요구사항과 완료 기준을 명확히 정의하기</li>
<li>민감정보와 접근 범위를 통제하기</li>
<li>변경 diff와 실행 명령을 검토하기</li>
<li>테스트 결과와 실제 동작을 최종 판단하기</li>
</ul>

## 발표자 체크 질문

1. 채팅형 AI와 에이전트형 도구의 가장 큰 차이는 무엇인가?
2. Claude Code가 코드를 수정한 뒤 반드시 해야 할 일은 무엇인가?
3. 어떤 작업부터 맡기는 것이 안전한가?
