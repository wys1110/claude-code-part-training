Git

Git은 되돌아갈 수 있는
저장 지점을 만든다

확인 → Commit → Push 순서로 공개 가능한 상태를 남깁니다.

Git 상태

git status는 지금 바뀐 것을 알려준다

git status
modified: index.html
modified: styles.css

의도한 파일만 바뀌었는지 확인합니다.

변경 내용

git diff는 실제로 바뀐 내용을 보여준다

git diff
- <h2>임시 제목</h2>
+ <h2>개인 업무 포트폴리오</h2>

예상한 내용만 추가·수정되었는지 봅니다.

Commit

Commit은 설명이 붙은
저장 지점이다

git commit -m "feat: add portfolio page"
확인한 변경만 기록

Push

Push는 확인한 저장 지점을
GitHub로 보낸다

git push origin main

08 · 검토 완료

Diff와 변경 파일이 맞으면
시연 1차 완료다

git status --short
?? index.html · ?? styles.css · ?? app.js

git diff --no-index /dev/null index.html
git diff --no-index /dev/null styles.css
git diff --no-index /dev/null app.js

git diff --no-index --check /dev/null index.html
git diff --no-index --check /dev/null styles.css
git diff --no-index --check /dev/null app.js
+ 줄을 검토 · 공백 오류 출력 없음

exit 1은 파일 차이를 알리는 정상 결과입니다. 이제 사람이 내용을 검토하고 Commit·Push·공개 여부를 결정합니다.
