# GitHub Pages Starter

Solution PE팀 Staff 교육의 **GitHub 저장소 생성 → 첫 HTML Commit → Pages 활성화** 실습용 공개 예제입니다.

## 사용 방법

1. GitHub에서 `my-first-pages-<github-id>` Public 저장소를 만듭니다.
2. `Add a README file`을 활성화합니다.
3. 저장소 루트에 `index.html`을 생성합니다.
4. 이 폴더의 `index.html` 내용을 붙여 넣습니다.
5. 제목, 설명, 학습 목표만 공개 가능한 문장으로 수정합니다.
6. Commit 후 Settings → Pages에서 `Deploy from a branch`, `main`, `/ (root)`를 선택합니다.
7. Actions 성공과 공개 URL을 확인합니다.
8. HTML을 한 번 수정해 재배포를 확인합니다.

## 저장소 입력 예시

```text
Repository: my-first-pages-jayden
Description: My first GitHub Pages practice site
Visibility: Public
Initialize: Add a README file
```

## 업로드 금지

- 회사 코드·문서·업무 로그
- 고객·임직원 정보
- API 키·비밀번호·인증 토큰
- 비공개 URL·내부 이미지
- 전화번호·이메일 등 개인정보

Public 저장소와 Pages 사이트는 인터넷에서 누구나 볼 수 있습니다. 민감한 자료를 Commit한 뒤 삭제하거나 Private으로 전환하는 방식은 안전한 대응이 아닙니다.
